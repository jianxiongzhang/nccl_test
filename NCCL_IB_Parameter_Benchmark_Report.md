# NCCL InfiniBand 参数对比测试报告

## 1. 测试目的

对比三组环境变量组合对 **AllReduce** 与 **AllToAll** 的影响，涉及：

| 变量 | 含义（简述） |
|------|----------------|
| `NCCL_IB_ADAPTIVE_ROUTING` | IB 自适应路由 |
| `NCCL_IB_OOO_RQ` | 乱序接收队列相关能力 |
| `NCCL_IB_RECEIVER_SIDE_MATCHING_SCHEME` | 接收端匹配策略 |
| `NCCL_IB_PREPOST_RECEIVE_WORK_REQUESTS` | 预投递接收 WR |

## 2. 环境与固定条件

| 项 | 说明 |
|----|------|
| **规模** | 2 节点 × 8 GPU = **16 ranks**（R6KD-CX8aaS-GPU-11 / GPU-12） |
| **GPU** | NVIDIA RTX 6000D |
| **NCCL** | 2.30.1+cuda13.0（nccl-tests 2.18.2，headers/library 23001） |
| **传输** | `UCX_TLS=ib`，`NCCL_SHM_DISABLE=1`，`NCCL_P2P_LEVEL=PHB`，`NCCL_NET_PLUGIN=none` |
| **网卡** | `NCCL_IB_HCA` 指定 8× mlx5（每节点 8 口），`NCCL_NETDEVS_POLICY=MAX:1` |
| **缓冲** | `NCCL_BUFFSIZE=16777216` |
| **测试** | `all_reduce_perf` / `alltoall_perf`，`-b 1k -e 8G -f 2 -g 1`，20 次迭代，校验通过 |

## 3. 三组参数配置

| 配置 | `ADAPTIVE_ROUTING` | `OOO_RQ` | `RECEIVER_SIDE_MATCHING` | `PREPOST_RECV_WR` |
|------|-------------------|----------|--------------------------|-------------------|
| **A（全关）** | 0 | 0 | 0 | 0 |
| **B（仅开 AR）** | 1 | 0 | 0 | 0 |
| **C（全开）** | 1 | 1 | 1 | 1 |

说明：B 与 A 的差异仅为 **自适应路由**；C 在 B 基础上再打开 **OOO RQ + 接收端匹配 + 预投递接收**。

## 4. nccl-tests 汇总指标

工具输出的 **Avg bus bandwidth**（全量程平均，含小消息）。**B/C 相对 A** 为 \((X/A - 1) \times 100\%\)，保留两位小数。

| 指标 | A (GB/s) | B (GB/s) | B 相对 A | C (GB/s) | C 相对 A |
|------|---------:|---------:|---------:|---------:|---------:|
| AllReduce Avg busbw | 20.8116 | 20.7852 | −0.13% | 20.8979 | +0.41% |
| AllToAll Avg busbw | 21.3866 | 20.7567 | −2.94% | 21.4657 | +0.37% |

## 5. 大消息峰值（8 GiB，out-of-place busbw）

更贴近网络饱和能力。**B/C 相对 A** 计算方式同第 4 节。

| 集体 | A (GB/s) | B (GB/s) | B 相对 A | C (GB/s) | C 相对 A |
|------|---------:|---------:|---------:|---------:|---------:|
| AllReduce @ 8G | 49.55 | 49.53 | −0.04% | 49.65 | +0.20% |
| AllToAll @ 8G | 40.81 | 40.25 | −1.37% | 40.98 | +0.42% |

## 6. 简要结论

1. **仅开启 `NCCL_IB_ADAPTIVE_ROUTING`（B）**：AllReduce 平均与峰值与 A 几乎持平；**AllToAll 平均总线带宽明显下降**（约 3%），8G 峰值也略低。在本拓扑与负载下，AR 单独打开对 AllToAll 不利或带来额外抖动，需结合交换机 AR 配置与是否真有等价多路径再判断。
2. **同时打开 OOO RQ、接收端匹配、预投递 + AR（C）**：AllReduce / AllToAll 的 **平均总线带宽与 8G 峰值均略优于 A**，说明这些接收侧与队列相关选项与 AR 组合后，整体略好于「全关」和「仅 AR」。
3. **数值差异量级**：大消息 AllReduce 峰值相差约 **0.1–0.4 GB/s**（相对 ~49.5 GB/s 约 **0.2–0.8%**），属于小幅差异；结论应结合多次重复运行与置信区间更稳妥。
4. **正确性**：三组 `#wrong` 均为 0，`Out of bounds values : 0 OK`。

## 7. 建议（可选后续）

- 对 **B 与 C** 做 **拆分实验**（每次只改一个变量），以区分是 AR 还是 OOO/匹配/预投递各自的作用。
- 补充 **多轮重复** 与 **不同消息步进或 rank 数**，避免单次运行噪声。
- 记录交换机侧 **Adaptive Routing / ECMP** 是否与 `NCCL_IB_ADAPTIVE_ROUTING` 一致，便于解释 B 组 AllToAll 回落。
